1. 安装与启动

a. 在 FnOS 应用中心安装

1. 下载 fnytdlp.fpk
2. 进入飞牛NAS -> 应用中心 -> 手动安装 -> 选择 fpk 文件
3. 安装后应用中心启动应用
4. 首次使用点击右下角的"设置"图标，配置下载路径

b. 直接运行（开发调试）

cd /path/to/fnytdlp
TRM_PKGVAR=/tmp/fnytdlp-test PORT=19634 node ui/server.js

然后浏览器访问 http://localhost:19634

c. 版本号

应用页脚显示当前版本号，如 v0.2.0。

3. 下载视频

3.1 添加任务

1. 点击工具栏"+ 添加"或按键盘 N
2. 在弹窗中输入视频 URL（支持单行粘贴），例如：
   https://www.youtube.com/watch?v=xxxx
   https://www.bilibili.com/video/BVxxxx
   https://www.douyin.com/video/xxxx
3. 可选：点击"解析第一条"预览视频信息
4. 可选：选择 Cookie 认证（如果站点需要登录）
5. 可选：选择格式（默认使用设置中的全局格式）
6. 展开高级选项可配置输出文件名模板、SponsorBlock、元数据等
7. 点击"开始下载"

批量添加：每行一个 URL，批量提交。yt-dlp 会自动处理播放列表。

3.2 格式选择

使用默认    -> 跟随设置中的全局格式
bv*+ba/b   -> 最佳画质+音频（推荐）
bv*+ba     -> 最佳画质+最佳音频，分离下载后合并
b/b        -> 最佳（单文件）
worst      -> 最小体积，最低画质最快速度
4K/2K/1080p/720p/480p 限高 -> 限制最大分辨率
ba / ba/b  -> 仅音频（最佳/降级）
worstaudio -> 仅音频（最小）

也支持自定义 yt-dlp 格式表达式，如：
bv*[height<=1080][fps<=30]+ba

3.3 URL 解析

在解析弹窗中输入 URL 点击解析，可获取视频的标题、时长、上传者、
可用的格式列表（分辨率/编码/码率/大小）、是否支持字幕。

3.4 SponsorBlock 广告跳过

支持自动标记/跳过视频中的赞助片段，在新建任务高级选项中设置，
多个类别用逗号分隔：

sponsor       -> 广告/赞助段落
selfpromo     -> 作者自推广
intro         -> 片头介绍
outro         -> 片尾
interaction   -> 互动提醒
preview       -> 预告/下集预览
music_offtopic -> 非音乐内容

4. 设置选项

点击"设置"打开设置面板，分为 3 个标签页：

4.1 基本设置

下载路径          -> 视频保存目录（建议 /volX/1000/fnytdlp/）
同时下载任务数    -> 最多同时下载几个任务（默认 3）
分片并发数        -> 单个文件分片下载的线程数（默认 4）
默认格式          -> yt-dlp 格式表达式（默认 bv*+ba/b）
格式排序          -> 格式优先级排序（默认 res:1080,ext:mp4:m4a）
重试次数          -> 下载失败自动重试（默认 3）
HTTP 代理         -> 通过代理下载（如 Clash/V2Ray）

复选框选项：

嵌入元数据   -> 将标题/上传者/日期写入文件元数据
下载字幕     -> 自动下载字幕
自动字幕     -> 下载自动生成的字幕
缩略图       -> 下载视频缩略图
单视频       -> 不下载播放列表，只下当前视频
字幕嵌入     -> 字幕内嵌到视频文件
缩略图嵌入   -> 缩略图嵌入到视频文件
仅音频       -> 不下载视频，只下载音频
多音轨       -> 下载所有可用音轨
mtime        -> 文件修改时间设为视频发布时间

4.2 高级设置

限速            -> 限制下载速度（如 5M / 500K）
字幕语言        -> 指定字幕语言（如 zh-Hans,en）
音频格式        -> 仅音频模式下的输出格式（mp3/m4a/opus/flac）
音频质量        -> 0（最佳）~ 10（最差）
合并输出        -> 视频音频合并格式（mp4/mkv）
字幕转换        -> 字幕格式转换（srt/vtt/ass）
下载归档        -> 避免重复下载，已下载视频 ID 自动跳过
最大下载数      -> 限制一次最多下载多少个视频

4.3 过滤设置

起始日期    -> 只下载此日期后的视频（如 20260101 / today-1week）
截止日期    -> 只下载此日期前的视频
最小文件大小 -> 过滤掉小文件（如 50M）
最大文件大小 -> 过滤掉大文件（如 2G）
复杂过滤    -> yt-dlp match-filters 表达式

5. Cookie 认证

某些网站需要登录才能下载（如 YouTube 会员视频、B站大会员）。

5.1 获取 Cookie

1. 在浏览器安装 Get cookies.txt LOCALLY 扩展
2. 登录目标网站（如 youtube.com / bilibili.com）
3. 点击扩展 -> 导出 cookies.txt

5.2 导入 Cookie

1. 应用内点击 Cookie 按钮
2. 填写：名称（唯一标识，如 youtube / bilibili），
          网站域名（可选），
          Cookies 内容（粘贴 cookies.txt 全文）
3. 点击保存 Cookie

5.3 使用 Cookie

新建任务时，Cookie 认证下拉框选择对应的 Cookie 名称即可。
每个网站独立配置，互不干扰。

5.4 管理 Cookie

Cookie 管理界面会列出所有已保存的 Cookie，可逐个删除。

6. 任务管理

6.1 任务状态

下载中 -> 正在下载，显示实时速度 + 进度条
已完成 -> 下载成功，显示文件大小
出错   -> 下载失败，显示错误信息
已暂停 -> 手动停止，可重新启动

6.2 任务操作

单击任务卡   -> 查看详情（13 个字段）
点击删除图标 -> 删除任务（可选择"仅删除记录"或"删除记录+文件"）
点击停止图标 -> 停止下载
点击启动图标 -> 恢复下载

6.3 批量操作

勾选任务前的复选框，或勾选顶部"全选"：

取消   -> 取消全选
重试   -> 批量重试出错任务
停止   -> 批量停止下载中任务
启动   -> 批量启动已暂停任务
删除   -> 批量删除（需确认）

6.4 任务详情弹窗

点击任务卡弹出，包含 13 个字段：

ID         -> 任务唯一标识
标题       -> 视频标题
URL        -> 原始视频链接
文件名     -> 保存的文件名
文件大小   -> 总大小
已下载     -> 已下载大小
存储位置   -> 文件所在目录
状态       -> 当前状态
进度       -> 下载进度百分比
格式       -> 视频编码 + 分辨率 + 音频格式
创建时间   -> 任务添加时间
完成时间   -> 下载完成时间
错误信息   -> 如果出错显示错误原因

6.5 多流下载（视频+音频分离）

当 yt-dlp 下载视频和音频为两个独立流时：
- 任务列表显示两段子进度条：视频流 0-50%、音频流 50-100%
- 两部分都下载完成后自动合并
- 格式描述如：1080p HEVC.mp4 + 128k mp4a.m4a

6.6 常用操作路径

下一个视频 -> 按 N 键 -> 粘贴 URL -> 回车开始下载
不同站需登录 -> 按 N 键 -> 粘贴 URL -> 选 Cookie -> 下载
只下音频   -> 设置里勾选"仅音频"或任务里选"仅音频"格式
批量播放列表 -> 设置里取消勾选"单视频" -> 粘贴播放列表 URL
清理已完成 -> 工具栏"清理" -> 确认
找任务     -> 搜索栏输入 URL 或文件名片段

8. 常见问题

Q: 下载速度很慢？

尝试以下设置：
1. 分片并发数：调大到 8-16（设置 -> 基本）
2. 同时下载任务数：调低到 1-2（任务少时单任务速度更高）
3. 限速：确保没有设置限速
4. 代理：如果需要用代理，在设置 -> 基本填 HTTP 代理地址

Q: 下载 B 站视频需要大会员？

1. 浏览器登录 B 站大会员账号
2. 用 Get cookies.txt LOCALLY 扩展导出 cookies
3. 应用内 Cookie -> 保存 -> 名称填 bilibili
4. 新建任务，Cookie 认证选择 bilibili

Q: 下载后文件在哪里？

设置 -> 基本 -> 下载路径中配置的目录。默认在应用数据目录下的
data/downloads/。

Q: 为什么有些视频只下了音频或低画质？

可能原因：
1. 检查默认格式设置，确保不是 ba（仅音频）或 worst（最小体积）
2. 某些视频站限制画质（如非会员限 720p）
3. 任务级格式选择覆盖了全局设置

Q: 支持哪些视频站？

yt-dlp 支持 1872+ 站点，国内常见的有：
Bilibili/哔哩哔哩、抖音/TikTok、微博、西瓜视频、好看视频、
AcFun、腾讯视频、爱奇艺、优酷、网易云音乐

国际常见的有：
YouTube、Twitter/X、Twitch、Vimeo、Dailymotion、
Facebook/Instagram、Reddit、SoundCloud、Bandcamp、PBS/NPR

Q: 如何开启下载归档？

设置 -> 高级 -> 下载归档，填 archive.txt。已下载的视频 ID 会
被记录，下次遇到相同视频自动跳过，避免重复下载。

Q: 应用打不开/报错？

1. 确认已在 fnOS 应用中心启动应用
2. 在应用中心重启应用（不是刷新页面，需要重启进程）
3. 检查 /vol3/@appdata/fnytdlp/info.log 查看错误日志
